# Freshdesk Triage Cron Cards v2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run both Freshdesk triage views unattended on Hermes, classify with a tool-restricted Agent, and send deterministic DingTalk cards without enabling unattended Freshdesk writes.

**Architecture:** Hermes cron runs a Python wrapper in `--no-agent` mode. One shared standard-library driver fetches the existing GET-only inspector output, invokes a nested Hermes oneshot with only the harmless `todo` toolset, validates its JSON, orders and renders tables, and sends a DWS stream card. The existing supervised two-way assignment helper remains separate and unchanged.

**Tech Stack:** Python standard library, existing Freshdesk inspector, Hermes oneshot, DWS CLI v1.0.52+, assertion-based validation scripts.
